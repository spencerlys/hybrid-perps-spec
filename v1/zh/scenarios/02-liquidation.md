# 清算场景

---

## SC-LQ-001：逐仓 INTERNAL 仓位清算

**前置条件：**
- BTC INTERNAL 多头：size=0.1 BTC，entry_price=$80,000，杠杆 10x，逐仓模式
- 保证金 $800，维持保证金率 0.4%（从 HL meta API 获取）

**清算价格计算：**
```
清算价 = 80,000 × (1 - 800/8,000 + 0.004) = 80,000 × 0.904 = $72,320
```

**操作步骤：**
1. HL 标记价格下跌至 $72,300（低于清算价 $72,320）
2. XBIT 清算引擎检测触发（目标 < 1s）
3. 按 HL mark price $72,300 内部结算

**预期结果：**
- 用户保证金归零（$800）
- 客损分配：$640（80%）→ 平台利润，$160（20%）→ 风险准备金
- 仓位状态 → LIQUIDATED
- 推送清算通知给用户
- 其他仓位不受影响

## SC-LQ-002：逐仓 HYPERLIQUID 仓位清算

**前置条件：**
- ETH HYPERLIQUID 空头：size=5 ETH，entry_price=$4,000，杠杆 5x，逐仓
- 保证金 $4,000，维持保证金率 0.5%

**清算价格计算：**
```
清算价 = 4,000 × (1 + 4,000/20,000 - 0.005) = 4,000 × 1.195 = $4,780
```

**操作步骤：**
1. HL 标记价格上涨至 $4,790（超过清算价 $4,780）
2. XBIT 清算引擎检测触发
3. 向 HL 发送市价平仓指令（size=5 ETH）
4. 等待 HL 回执 close_price

**预期结果：**
- HL 返回 close_price（可能有滑点，如 $4,792）
- 已实现 PnL 按 close_price 计算
- 若 HL 实际亏损 > XBIT 预算 → 差额从风险准备金扣除
- 用户保证金归零
- HL 平台合并仓位减少对应 ETH 数量

## SC-LQ-003：全仓模式 — 账户级清算触发

**前置条件：**
- 用户全仓账户：钱包余额 $5,000
  - BTC 多头 INTERNAL：notional=$10,000，未实现 PnL = -$4,000（BTC 跌）
  - ETH 空头 HYPERLIQUID：notional=$8,000，未实现 PnL = -$500（ETH 涨）
  - 维保率：BTC 0.4%，ETH 0.5%

**计算过程：**
```
账户权益 = $5,000 + (-$4,000) + (-$500) = $500
全仓维持保证金需求 = $10,000×0.4% + $8,000×0.5% = $40 + $40 = $80

账户权益 $500 > $80 → 暂未触发（价格继续下跌...）
账户权益降至 $80 → 触发清算
```

**预期结果：**
- 先平亏损最大的仓位（BTC INTERNAL，亏损 -$4,000）
- BTC INTERNAL → L3 内部结算
- 重新计算账户权益，若已恢复 > 剩余维保要求 → 停止清算
- 若仍不足 → 继续平 ETH HYPERLIQUID 仓位

## SC-LQ-004：全仓清算 — 连续平仓直到账户清零

**前置条件：**
- 账户权益已降至维持保证金需求
- 持有 3 个亏损仓位

**操作步骤：**
1. 平最大亏损仓位 → 重新计算账户权益
2. 权益仍 ≤ 维保需求 → 平第二个仓位
3. 权益仍 ≤ 维保需求 → 平第三个仓位
4. 所有仓位平完，账户归零

**预期结果：**
- INTERNAL 仓位 → 内部结算，客损 80/20 分配
- HYPERLIQUID 仓位 → 向 HL 发平仓指令，等回执
- 每平完一个后立即重新判断是否继续
- 用户账户余额归零，推送清算通知

## SC-LQ-005：混合模式 — 逐仓爆仓不影响全仓仓位

**前置条件：**
- 用户账户：逐仓 BTC 多头 $500 保证金（INTERNAL）+ 全仓 ETH 多头（CROSS）
- 全仓可用余额 $8,000

**操作步骤：**
1. BTC 价格下跌，逐仓 BTC 仓位触发清算
2. BTC INTERNAL 仓位按 mark price 内部结算

**预期结果：**
- BTC 逐仓仓位 LIQUIDATED
- ETH 全仓仓位不受任何影响（独立保证金池）
- 全仓可用余额 $8,000 不变
- 仅推送 BTC 清算通知

## SC-LQ-006：资金费侵蚀导致清算价上移

**前置条件：**
- BTC 逐仓多头：entry=$80,000，保证金=$800，维保率 0.4%
- 持仓 30 天，累计资金费支出 $200（多头持续付出）

**清算价格对比：**
```
未考虑资金费：
  清算价 = 80,000 × (1 - 800/8,000 + 0.004) = $72,320

考虑资金费侵蚀：
  清算价 = 80,000 × (1 - (800-200)/8,000 + 0.004) = $74,320
```

**预期结果：**
- 实际清算价 $74,320（比未考虑资金费高 $2,000）
- 用户实际保证金已被资金费侵蚀 $200
- 每次资金费结算后系统自动更新清算价显示
