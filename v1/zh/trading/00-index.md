## 系统层次图

```
用户请求
  │
  ▼
L1 账户与资产层（余额校验、保证金计算）
  │
  ▼
L2 订单路由层（<5ms 路由决策）
  │
  ├─ INTERNAL ──► L3 平台对赌执行层（<10ms）
  │                    │
  └─ HYPERLIQUID ──► L4 HL 代理执行层（<50ms）
                         │
  ◄─────────────────────┤ HL 成交回执（fill_price）
  │
  ▼
L5 保证金与清算引擎（实时监控所有仓位）
  │
  ▼
L6 风控与敞口管理（净敞口聚合 + 对冲指令）
  │
  ▼
L7 市场数据层（HL mark price → 所有层）
  │
  ▼
L8 结算与对账层（PnL、资金费、三方对账）
  │
  ▼
L9 后台管理层（配置、仪表盘、审计）
```

## 关键数据流说明

| 数据 | 流向 | 频率 |
|------|------|------|
| HL 标记价格 | L7 → L5（清算检测） | 实时 |
| HL 标记价格 | L7 → L3（成交价格参考） | 实时 |
| HL 回执价格 | L4 → 仓位记录（entry_price 校正） | 每笔成交 |
| 清算信号 | L5 → L3 / L4 | 事件驱动 |
| 净敞口 | L3 → L6 | 实时聚合 |
| 对冲指令 | L6 → L4 | 阈值触发 |
