# 验收清单

本文档规定化永续期货平台（方案二）MVP各阶段的验收标准与检查清单。分为功能、风控、性能、运营四个维度，共100+个可打勾的验收项。

---

## 一、Phase 1 验收清单（基础接入）

### 1.1 账户与认证（T1负责）

- [ ] 用户可通过邮箱注册 (支持邮箱验证码)
- [ ] 用户可通过邮箱登录 (会话管理正常)
- [ ] 用户可通过MetaMask签名登录 (无邮箱)
- [ ] 用户可使用Google/Apple OAuth登录
- [ ] 用户资料编辑功能正常 (昵称/头像/KYC信息)
- [ ] 密码重置功能正常 (邮箱验证+新密码设置)
- [ ] Web2与Web3账户可关联 (同一用户同时支持两种登录)
- [ ] 会话安全: 支持2FA (TOTP)
- [ ] 会话安全: token过期时间<1小时
- [ ] 会话安全: 支持设备信任列表

### 1.2 钱包与资产（T1负责）

- [ ] 用户有独立钱包地址 (自动分配或导入)
- [ ] 钱包余额显示正确 (USDT/USDC/其他)
- [ ] 余额流水记录完整 (充值/提现/交易/手续费)
- [ ] 余额流水可按时间/类型筛选
- [ ] 总资产计算正确 (现金+持仓)
- [ ] 冻结金额显示准确 (提现中的金额冻结)
- [ ] 支持多币种 (USDT/USDC/USDT-Solana等)

### 1.3 TRON充值（T1负责）

- [ ] 生成TRON充值地址 (TRC20)
- [ ] 充值地址显示正确，长期有效
- [ ] 充值页面显示网络选择 (TRON主网)
- [ ] 充值金额输入验证 (最小$10, 最大$100K)
- [ ] 链上TRON转账 (扫链间隔<30秒)
- [ ] 19确认后自动入账 (不需手动操作)
- [ ] 充值历史记录完整 (TxHash/时间/金额/状态)
- [ ] 充值失败时显示清晰提示 (e.g. "地址黑名单")
- [ ] 支持多笔同时充值 (并行处理)

### 1.4 ETH充值（T1负责）

- [ ] 生成ETH充值地址 (支持ERC20)
- [ ] 充值页面支持选择币种 (USDT/USDC)
- [ ] 链上ETH转账扫链正常 (12确认)
- [ ] 12确认后自动入账
- [ ] ETH与TRON充值不冲突 (用户可同时充两种)

### 1.5 提现流程（T1负责）

- [ ] 用户可发起提现申请 (输入金额/地址/链)
- [ ] <$10K提现: 自动审核通过，5分钟内链上到账
- [ ] $10K-$100K提现: 需1名运营人员人工确认 (1小时内)
- [ ] >$100K提现: 需风控+运营2人审批 (2小时内)
- [ ] 提现地址管理: 可添加/删除/标记地址
- [ ] 提现地址白名单: 只能提现到预置地址 (可选策略)
- [ ] 提现手续费透明显示
- [ ] 提现历史记录完整 (状态/金额/TxHash)
- [ ] 提现被拒: 显示拒绝原因 (e.g. "金额超限")
- [ ] 重复提现防护: 同一地址10分钟内只能提一次

### 1.6 多链归集与热冷钱包（T1负责）

- [ ] 热钱包余额监控 (>$1M则自动转冷)
- [ ] 冷钱包到热钱包自动补资 (<$100K时)
- [ ] 跨链转账流程清晰 (TRON→ETH)
- [ ] 多链汇总展示 (总余额 = TRON+ETH+Solana)
- [ ] 钱包地址安全管理 (私钥不被暴露)

### 1.7 HL API中继（T3负责）

- [ ] REST API全量代理 (GET /v1/*, POST /v1/*)
- [ ] 请求格式自动转换 (如需)
- [ ] 响应格式正确 (JSON)
- [ ] 错误处理: HL错误信息透传给用户
- [ ] 请求超时处理 (>30秒超时返回错误)
- [ ] 请求去重 (同一请求不重复发送)
- [ ] API速率限制管理 (不超过HL限额)

### 1.8 HL WebSocket透传（T3负责）

- [ ] WebSocket连接正常建立
- [ ] 支持多个频道同时订阅 (prices, orderbook, trades)
- [ ] 价格数据实时推送 (<100ms)
- [ ] 订单簿数据实时推送
- [ ] K线数据实时推送
- [ ] 心跳管理: 服务端定期心跳，客户端超时重连
- [ ] 断线自动重连 (指数退避)
- [ ] 多个用户的WS独立管理 (互不影响)

### 1.9 市场数据展示（T3+T6负责）

- [ ] 价格行情页面显示BTC/ETH/其他主流币
- [ ] 实时价格刷新 (<500ms延迟)
- [ ] K线图显示 (1min/5min/15min/1h/4h/1d)
- [ ] 成交量显示
- [ ] 资金费率显示 (可点击查看历史)
- [ ] 24h涨跌幅显示
- [ ] 支持搜索币种

### 1.10 后台框架与RBAC（T6负责）

- [ ] 后台登录 (Super Admin/Risk Manager/Operations/Data Analyst)
- [ ] 角色权限隔离 (各角色只能看/做自己权限内的操作)
- [ ] 四种角色功能完整划分:
  - [ ] Super Admin: 全部功能 + 系统参数修改
  - [ ] Risk Manager: 风控配置 + 路由模式 + 对冲阈值
  - [ ] Operations: 提现审核 + 用户查询 (只读)
  - [ ] Data Analyst: 数据报表 + 审计日志 (只读)
- [ ] 后台操作审计日志 (谁操作了什么，什么时候)

### 1.11 用户查询页面（T6负责）

- [ ] 后台可按用户ID/邮箱查询用户信息
- [ ] 显示用户资产明细 (充值/提现/交易历史)
- [ ] 显示用户钱包地址 + 绑定时间
- [ ] 可导出用户列表 (Excel)
- [ ] 支持时间范围筛选

### 1.12 基础设施（T3负责）

- [ ] PostgreSQL数据库部署正常，备份策略完善
  - [ ] 日备份（至少保留7天）
  - [ ] 主从复制正常
  - [ ] 故障恢复测试通过
- [ ] Redis集群部署正常，支持缓存预热
- [ ] 消息队列部署正常 (RabbitMQ/Kafka)
  - [ ] 支持消息持久化
  - [ ] 死信队列配置正确
- [ ] Docker容器化部署
- [ ] CI/CD流程完整 (git push → 自动测试 → 自动部署)
  - [ ] Pre-commit hook检查代码格式
  - [ ] 单元测试>80%通过率才允许merge
  - [ ] 自动部署到测试环境
- [ ] Prometheus监控部署
  - [ ] 收集关键指标 (API延迟/错误率/QPS)
  - [ ] 告警规则配置 (>5s延迟、>1%错误率)
- [ ] Grafana仪表盘配置
  - [ ] 实时系统状态看板
  - [ ] 历史趋势看板

### 1.13 Phase 1 验收通过标准

- [ ] 所有功能项全部通过 ✓

- [ ] 用户反馈无关键问题
- [ ] 数据一致性验证通过 (用户资产对账误差<0.01%)

## 二、Phase 2 验收清单（路由+对冲）

### 2.1 订单路由引擎（T2负责）

- [ ] 路由模式切换正常 (HL_MODE → NORMAL_MODE → BETTING_MODE)
- [ ] NORMAL_MODE: ≤$10K订单 → 标记route=INTERNAL
- [ ] NORMAL_MODE: >$10K订单 → 标记route=HL
- [ ] BETTING_MODE: ≤$50K订单 → 标记route=INTERNAL
- [ ] BETTING_MODE: >$50K订单 → 标记route=HL
- [ ] 强制路由HL条件生效:
  - [ ] 波动>5%/小时 → HL_MODE自动切换
  - [ ] HL延迟>500ms → HL_MODE自动切换
  - [ ] 恢复后自动切回前一模式
- [ ] 路由决策延迟 <5ms P99
- [ ] 路由日志完整 (订单ID/金额/决策/理由)

### 2.2 订单生命周期管理（T2负责）

- [ ] 创建订单: 订单状态=PENDING
- [ ] 订单入库: 状态=CREATED, 分配订单ID
- [ ] 订单回执: 获得成交价/成交量
- [ ] 部分成交处理: 累计成交量，记录多笔
- [ ] 订单完全成交: 状态=FILLED
- [ ] 订单取消: 状态=CANCELLED
- [ ] 订单异常: 状态=ERROR，记录错误信息
- [ ] 订单重试机制: 失败后自动重试 (3次)
- [ ] 订单查询: 支持按订单ID/币种/时间范围查询

### 2.3 HL代理执行（T3负责）

- [ ] 订单转发到HL: 立即发送
- [ ] HL订单ID映射: 内部订单←→HL订单一一对应
- [ ] HL回执正确处理:
  - [ ] 成交价格记录
  - [ ] 成交数量记录
  - [ ] 成交时间记录
- [ ] 价格校正: 回执价 vs 用户请求价差距<1bp
- [ ] 仓位同步: HL仓位每5秒同步一次
- [ ] 仓位一致性: 内部记录=HL实际, 误差<0.01%
- [ ] 订单转发延迟 <50ms P99
- [ ] HL连接稳定性: 断连恢复时间<5分钟

### 2.4 预交易风控（T4负责）

- [ ] 余额检查: 余额不足→拒绝下单, 返回"余额不足"
- [ ] 币种检查: 币种暂停交易→拒绝下单, 返回"币种不可用"
- [ ] 杠杆检查: 杠杆>最大允许值→拒绝, 返回"杠杆过高"
- [ ] 异常订单检查: 金额为0 / 负数→拒绝
- [ ] 地址黑名单检查: 黑名单用户→拒绝
- [ ] 风控检查延迟 <10ms

### 2.5 净敞口实时计算（T4负责）

- [ ] 计算公式正确: Net_Exposure = Σ(Long_INTERNAL) - Σ(Short_INTERNAL)
- [ ] 支持多币种敞口分离 (BTC敞口/ETH敞口分别计算)
- [ ] 每有新订单/平仓时，净敞口实时更新
- [ ] 敞口计算精度 ≥ 99.5%
- [ ] 敞口计算延迟 <1s
- [ ] 可在后台查看实时敞口

### 2.6 对冲引擎（T4负责）

- [ ] 对冲阈值配置正确:
  - [ ] <$100K: 无对冲
  - [ ] $100K-$500K: 50%对冲
  - [ ] $500K-$1M: 80%对冲
  - [ ] >$1M: 80%对冲+停止新对赌订单
- [ ] 对冲指令生成: 符合阈值规则
- [ ] 对冲指令转发到HL: 立即执行
- [ ] 对冲仓位追踪: HL对冲仓位与指令一致
- [ ] 对冲仓位减仓: 敞口降低后自动减仓
- [ ] 对冲止损: 对冲仓位浮亏>5%自动平仓
- [ ] 对冲引擎延迟 <10ms (指令生成)

### 2.7 风险准备金（T4负责）

- [ ] 准备金初始化: 初始金额设为$500K (管理员手动转入)
- [ ] 准备金入账: 每笔用户亏损, 20%进准备金
  - [ ] 例: 用户亏损$100 → 准备金+$20
- [ ] 准备金出账: 用户资产异常时兜底
- [ ] 准备金余额监控: 实时显示在后台
- [ ] 准备金告警:
  - [ ] 余额<$500K → 黄色告警
  - [ ] 余额<$200K → 红色告警+暂停对赌
- [ ] 准备金日志: 所有入出账可查

### 2.8 日净亏损熔断（T4负责）

- [ ] 日净亏损定义: 当日所有用户INTERNAL亏损累计
- [ ] 熔断阈值: $500K
- [ ] 熔断触发: 日净亏损≥$500K → 暂停所有新对赌订单
  - [ ] 提示文案: "对赌额度今日已用尽，请明天重试"
- [ ] 熔断解除: 次日凌晨自动解除
- [ ] 熔断状态显示: 后台/前端都可见

### 2.9 偏差监控（T5负责）

- [ ] 定义偏差: 平台计算结果 vs HL实际结果的差异
  - [ ] 例: 平台计算用户持有BTC 10, HL实际 10.05 → 偏差0.05
- [ ] 偏差检测周期: 每10秒检查一次
- [ ] 偏差告警阈值:
  - [ ] 偏差>0.1% → 黄色告警
  - [ ] 偏差>1% → 红色告警(暂停交易)
- [ ] 偏差记录: 保存所有偏差事件(时间/原因/调整方式)
- [ ] 偏差兜底: 若无法追查原因，用准备金调整用户资产

### 2.10 后台控制页面（T6负责）

- [ ] 路由模式切换页面:
  - [ ] 显示当前模式
  - [ ] 支持一键切换 (HL_MODE / NORMAL_MODE / BETTING_MODE)
  - [ ] 显示切换历史
- [ ] 路由阈值调整页面:
  - [ ] 显示normal_threshold ($10,000)
  - [ ] 显示betting_threshold ($50,000)
  - [ ] 支持实时修改，无需重启
- [ ] HL账户状态页面:
  - [ ] 显示HL账户保证金率
  - [ ] 显示HL持仓 (币种/方向/数量/均价)
  - [ ] 显示HL钱包余额
  - [ ] 刷新频率<5秒

### 2.11 Phase 2 灰度验收标准

- [ ] 所有功能项全部通过 ✓
- [ ] 7天内零故障 (无P0/P1告警)
- [ ] 路由决策准确率 ≥ 99.9% (无错误路由)
- [ ] HL代理执行精度 ≥ 99.9% (成交价偏差<1bp)
- [ ] API可用性 ≥ 99.95%
- [ ] 风控计算精度 ≥ 99.5% (敞口/准备金)
- [ ] 无用户资产损失事件

## 三、Phase 3 验收清单（对赌上线）

### 3.1 对赌执行引擎（T2负责）

- [ ] 市价单执行:
  - [ ] 成交延迟 <10ms
  - [ ] 成交价格=当前最新价±0.5bp
  - [ ] 成交数量=用户请求数量
- [ ] 限价单队列:
  - [ ] 限价单进入队列
  - [ ] 当HL价格达到限价时自动成交
  - [ ] 成交价=限价 (不滑点)
  - [ ] 限价单超时取消: 7天未成交自动取消
- [ ] 对赌方式: 平台作为用户的对手方
  - [ ] 用户买入 → 平台做空 (自动创建)
  - [ ] 用户卖出 → 平台做多 (自动创建)
- [ ] 对赌仓位创建: 用户成交的同时平台仓位自动创建
- [ ] 对赌仓位查询: 用户可看到平台对手方仓位
- [ ] 对赌收益: 用户亏损时平台收益，用户盈利时平台亏损

### 3.2 内部限价单队列（T2负责）

- [ ] 限价单队列管理: 支持FIFO或优先级队列
- [ ] 价格触达检测: 每秒检查HL最新价格，触达则执行
- [ ] 触达执行延迟 <1s
- [ ] 部分触达处理: 可成交数量<订单数量时，先成交可成交部分
- [ ] 队列可视化: 后台显示当前等待成交的限价单

### 3.3 保证金计算引擎（T4负责）

- [ ] 逐仓保证金(Isolated):
  - [ ] 公式: 保证金比率 = 持仓价值 / 保证金
  - [ ] 维保率分级: 与HL一致
    - [ ] 初始保证金率: 按杠杆倍数反推 (10x杠杆=10%初始保证金率)
    - [ ] 维保率: 按风险等级分级 (BTC 3.6%, ETH 4.8% etc.)
  - [ ] 逐仓保证金计算精度 ≥ 95% (与HL对比)
- [ ] 全仓保证金(Cross):
  - [ ] 公式: 保证金比率 = 账户净值 / 总维保
  - [ ] 全仓保证金计算精度 ≥ 95%
- [ ] 浮亏实时计算:
  - [ ] 每秒更新浮亏
  - [ ] 浮亏=当前价格仓位 - 开仓价值

### 3.4 统一清算引擎（T4负责）

- [ ] 清算触发条件检测:
  - [ ] 逐仓: 保证金率 ≤ 维保率 → 触发
  - [ ] 全仓: 账户保证金率 ≤ 账户维保率 → 触发
  - [ ] 检测延迟 <1s (从触发到执行)
- [ ] 清算执行（逐仓）:
  - [ ] 直接平仓，金额和方向取决于仓位
  - [ ] 平仓价格: 使用当前市价 ± 0.5bp
  - [ ] 平仓完成: 仓位归零，保证金也归零
- [ ] 清算执行（全仓）:
  - [ ] 按亏损仓位排序，逐个平仓直到账户恢复
  - [ ] 平仓顺序: 最大亏损优先
  - [ ] 部分平仓: 若平仓部分仓位后账户恢复，停止清算
  - [ ] 全部平仓: 若无法恢复，平仓所有仓位
- [ ] 清算日志: 记录清算触发价/执行价/回收金额
- [ ] 清算失败处理: 若清算失败，转为人工审核

### 3.5 HL清算同步（T4负责）

- [ ] HL仓位清算检测: 监听HL清算事件
- [ ] 清算后处理: 发送平仓指令给HL，平仓完成后更新内部状态
- [ ] 清算对账: 清算后核对HL仓位和内部记录一致

### 3.6 HL计算精度复刻（T4负责）

- [ ] 维保率分级: 与HL完全一致
  - [ ] 获取HL最新维保率表(每小时同步一次)
  - [ ] 应用到清算计算
- [ ] 清算触发价格: 复刻HL的mark price计算
- [ ] 清算精度验证: 对>100笔清算事件，与HL对比
  - [ ] 清算触发价差距<0.5%
  - [ ] 清算执行结果差距<0.1%

### 3.7 完整PnL结算（T5负责）

- [ ] PnL定义: P&L = 成交收入 - 开仓成本 ± 资金费 - 手续费
- [ ] PnL计算时机: 仓位平仓时立即计算
- [ ] PnL精度: ≥ 99.99% (与HL对比)
- [ ] 浮动PnL: 实时显示 (基于当前价格)
- [ ] 已实现PnL: 显示已平仓仓位的PnL总和
- [ ] PnL查询: 用户可按仓位/币种/时间范围查看

### 3.8 资金费结算（T5负责）

- [ ] 资金费定义: 8小时为一个结算周期，每个周期支付1次
- [ ] 资金费率来源: 从HL WebSocket获取
- [ ] 结算时机: 每天 UTC 0:00 / 8:00 / 16:00
- [ ] INTERNAL仓位: 平台承担资金费 (用户无需支付)
  - 例: 用户持有1 BTC Long，资金费率0.01%

## 四、风控验收清单

### 4.1 下单前风控

- [ ] 余额不足→拒绝下单 + "余额不足"提示
- [ ] 币种不在允许列表→拒绝下单 + "币种暂停"提示

### 4.2 敞口控制

- [ ] 单币种净敞口 >$1M → 自动暂停该币种新对赌订单
  - [ ] 但已有订单继续执行
  - [ ] HL路由订单正常执行
- [ ] 敞口监控延迟 <5s

### 4.3 对冲账户保护

- [ ] 对冲账户保证金率 >300% (风险可控)
  - [ ] <300% → 黄色告警(减仓提醒)
  - [ ] <200% → 红色告警(自动减仓)
- [ ] 对冲杠杆 ≤5x (不超过设定值)

### 4.4 准备金监控

- [ ] 准备金 ≥ $500K: 正常状态
- [ ] 准备金 $200K-$500K: 黄色告警，缩减对赌数量
- [ ] 准备金 <$200K: 红色告警，暂停所有对赌
- [ ] 告警延迟 <1分钟

### 4.5 HL账户保护

- [ ] HL账户保证金率 >500% (保守目标)
  - [ ] <300% → 黄色告警
  - [ ] <150% → 红色告警+暂停HL开仓
- [ ] 若低于150%，HL账户有被清算风险 (平台灾难)
  - [ ] 必须立即补资金 或 平仓对冲仓位

### 4.6 市场风险防护

- [ ] 波动率>5%/小时→强制路由HL (关闭对赌)
- [ ] HL连接延迟>500ms→强制路由HL (关闭对赌)
- [ ] 两个条件恢复后自动切回NORMAL_MODE

### 4.7 清算风控

- [ ] 清算检测延迟 <1s (关键KPI)
- [ ] 清算执行精度 ≥ 99.8%
- [ ] 清算失败时发送P0告警

### 4.8 日净亏损熔断

- [ ] 日净亏损 >$500K → 暂停新对赌
- [ ] 次日自动解除
- [ ] 告警延迟 <1分钟

### 4.9 所有风控告警

- [ ] P0告警: Slack + PagerDuty + 电话
- [ ] P1告警: Slack + 邮件
- [ ] 告警内容清晰: 告警名称 / 触发值 / 建议动作
